This guide outlines the final steps for deploying the **NexLayer Operations System** to production.

## 1. Backend: Render (FastAPI)
Deploy the `backend/` directory as a **Web Service**.

### Configuration:
- **Environment**: Docker (Render will automatically detect the `Dockerfile` in the root).
- **Port**: 8080

### Environment Variables:
| Variable | Value |
| :--- | :--- |
| `FIREBASE_SERVICE_ACCOUNT` | The content of your `serviceAccountKey.json` file. |

> [!TIP]
> You can paste the JSON content directly into the Render environment variable field.

---

## 2. Frontend: Vercel (React)
Deploy the root project directory to **Vercel**.

### Configuration:
- **Framework Preset**: Vite
- **Root Directory**: `./` (or just leave default)
- **Build Command**: `npm run build`
- **Output Directory**: `dist`

### Environment Variables:
| Variable | Value |
| :--- | :--- |
| `VITE_API_BASE_URL` | Your Render Web Service URL (e.g., `https://nexlayer-backend.onrender.com`) |

---

## 3. Database & Auth: Firebase
Ensure your configuration is synchronized.

### Firestore Rules:
Deploy your rules to Firebase if you haven't already:
```bash
firebase deploy --only firestore:rules
```

---

## Final Verification Flow
1.  **Deploy Backend first** and copy the Render URL.
2.  **Deploy Frontend** and set the `VITE_API_BASE_URL` variable.
3.  **Test Guest Request**: Fill out the contact form and verify it appears on your CEO dashboard.
4.  **Test CEO Approval**: Approve the request and assign a team member.
5.  **Test Member Reporting**: Log in as a team member and submit a daily report.
